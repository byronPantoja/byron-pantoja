# Building an Interactive Salon Landing Page with NextJS in One Session

The first project in my [10-day portfolio sprint](/dispatch/10-day-portfolio-sprint-plan) was supposed to take three days. It shipped in one focused session. Here's how I built Serenity Studio — a multi-page wellness studio website with an interactive booking calendar, a journal with full blog articles, and a membership teaser — and what I learned about scope management along the way.

**Live site:** [mockup-salon.vercel.app](https://mockup-salon.vercel.app)  
**Source:** [github.com/byronPantoja/mockup-salon](https://github.com/byronPantoja/mockup-salon)

## What I Built

Serenity Studio is a fictional wellness/salon business site. It's not a single landing page — it's a complete, multi-page website:

- **Home page** — hero section, four service cards with pricing, about section, testimonial cards, interactive booking calendar, and contact form
- **/treatments** — a catalog of seven treatments, each with an accordion that expands to reveal an inline booking widget
- **/journal** — a blog index with three full articles, each on its own dynamic route with complete content
- **/membership** — a "coming soon" teaser page with email capture

The booking calendar is the star component. It renders a dynamic month view with clickable dates, time slot selection, and a confirmation flow. It appears on the home page as a standalone section AND inside each treatment's accordion on the treatments page.

## The Technical Decisions

**NextJS 14 with App Router** was the obvious choice — server components for the static pages, client components for interactive elements like the booking calendar and contact form. TypeScript throughout for type safety.

**Tailwind CSS** for styling. I chose a warm, premium palette — soft neutrals, sage green accents, and serif/sans-serif font pairing — to match the wellness industry. The design direction was defined before writing any code.

**No backend.** The booking calendar doesn't connect to Mindbody or any scheduling API. The contact form shows a success state but doesn't send emails. This was a deliberate scope decision — the portfolio demonstrates UI capability, not API integration.

## The Component That Matters Most

For anyone building a portfolio, the booking calendar is the kind of component that separates you from other applicants. Here's why:

Most landing page mockups are static. They show you can arrange text and images. An interactive calendar with date selection, time slot rendering, and state management shows you can build **application-level UI** — the kind of work employers actually need done.

The calendar uses `useState` to track the selected date and renders time slots conditionally. Clicking a date updates the availability panel. Clicking "Continue" shows a confirmation state. It's not complex code — but the polished result looks like production software.

## The Audit Process

After the initial build, I ran a thorough audit before moving on. This is where AI helped most — I pasted the deployed URL into Claude, and it crawled every page and flagged issues I would have missed:

- The booking calendar was hardcoded to November 2024 (16 months in the past)
- Footer links pointed to `#` instead of real pages
- Three treatment images were reused across seven treatments
- The contact form had no feedback state on submit

Each fix was small, but collectively they're the difference between "this person ships clean work" and "this person leaves loose ends." I turned the audit into a Claude Code prompt, ran it, and had everything fixed in about an hour.

## What I'd Do Differently

If I were building this for a real client (not a portfolio), I'd integrate with a scheduling API like Calendly or Cal.com instead of mocking the booking flow. I'd also add a CMS for the journal articles — right now they're hardcoded, which is fine for a mockup but doesn't scale.

The key insight: knowing what NOT to build is as important as knowing what to build. The Mindbody API integration would have taken 2-3 days and proven the same skill (API integration) that the Shopify store already covers in my portfolio. Scope discipline kept me on schedule.

## Key Takeaway for Other Developers

If you're building a portfolio to land remote work, your landing page mockup needs one interactive component that goes beyond static content. A booking calendar, a multi-step form, a filtering system — something that shows state management and user interaction. That's what employers click on. That's what makes them think "this person can handle our actual project."
